  Вот полный план разработки Time Lapse Screen Recorder на Electron:
Название "ZradaLog", мемное "Зрада" и логгирование.
Программа позиционируется в основном как рекордер выполенной работы в доказательство.

---

## 1. Стек технологий

| Компонент | Технология | Назначение |
|-----------|-----------|------------|
| **Фреймворк** | Electron 28+ | Основа приложения |
| **UI-фреймворк** | React 18 + TypeScript | Интерфейс |
| **Стилизация** | Tailwind CSS | Современный дизайн |
| **Сборка** | Vite | Быстрая компиляция |
| **Упаковка** | electron-builder | .exe, .dmg, .AppImage |
| **Захват экрана** | `desktopCapturer` (Electron API) | Выбор источника |
| **Запись видео** | FFmpeg (бинарники) | Кодирование, time lapse |
| **Сравнение кадров** | FFmpeg `select=gt(scene\,X)` | Игнорирование похожих |
| **Хранение настроек** | electron-store | JSON-конфиг |
| **Горячие клавиши** | electron-localshortcut | Глобальные хоткеи |
| **Трей** | Electron Tray API | Минимизация в трей |

## 3. Функциональные модули

### 3.1 Главное окно (Main Process)

**Функции:**
- Создание `BrowserWindow` с размерами 900×600
- Загрузка dev-server (dev) или index.html (prod)
- Инициализация Tray, глобальных хоткеев
- Управление жизненным циклом приложения

**Особенности:**
- `nodeIntegration: false`, `contextIsolation: true` (безопасность)
- Preload-скрипт как единственный мост между main и renderer

### 3.2 IPC Коммуникация (Preload → Main)

| Канал (channel) | Направление | Данные |
|-----------------|-------------|--------|
| `recorder:start` | Renderer → Main | `{ sourceId, fps, threshold, outputPath }` |
| `recorder:stop` | Renderer → Main | — |
| `recorder:status` | Main → Renderer | `{ isRecording, duration, frameCount }` |
| `config:get` | Renderer → Main | `key` |
| `config:set` | Renderer → Main | `{ key, value }` |
| `dialog:selectFolder` | Renderer → Main | — |
| `sources:get` | Renderer → Main | — |
| `sources:return` | Main → Renderer | `[{ id, name, thumbnail }]` |

### 3.3 Движок записи (RecorderEngine)

**Алгоритм:**

```
1. Получение sourceId от renderer
2. Запуск интервального таймера (1000 / fps ms)
3. Каждый тик:
   a. Получение кадра через desktopCapturer (или скриншот FFmpeg)
   b. Передача в FFmpeg через stdin pipe ИЛИ
   c. Сохранение во временную папку, сборка в конце
4. При остановке: корректное закрытие FFmpeg, финализация файла
```

**Режимы записи:**

| Режим | Описание | Когда использовать |
|-------|----------|------------------|
| **Direct FFmpeg** | Поток в FFmpeg через pipe | FPS ≥ 1, стабильный поток |
| **Frame Buffer** | Сохранение PNG → сборка в конце | FPS < 1, сложная обработка |
| **Scene Detection** | FFmpeg фильтр `select` | Игнорирование похожих кадров |

### 3.4 FFmpeg интеграция (FFmpegService)

**Команды для разных сценариев:**

**Time Lapse (низкий FPS):**
```bash
ffmpeg -f image2pipe -vcodec png -r 1 -i - 
       -c:v libx264 -preset medium -crf 23 
       -pix_fmt yuv420p output.mp4
```

**С игнорированием похожих кадров:**
```bash
ffmpeg -i input.mp4 
       -vf "select=gt(scene\,0.1),setpts=N/(25*TB)" 
       -c:v libx264 output.mp4
```

**Захват экрана напрямую (Windows):**
```bash
ffmpeg -f gdigrab -framerate 0.5 -i desktop 
       -c:v libx264 -preset fast output.mp4
```

**API сервиса:**
```typescript
class FFmpegService {
  startRecording(options: RecordingOptions): Promise<void>
  stopRecording(): Promise<string> // возвращает путь к файлу
  setFrameRate(fps: number): void
  setSimilarityThreshold(percent: number): void
  private spawnProcess(args: string[]): ChildProcess
}
```

### 3.5 Управление настройками (ConfigManager)

**Сохраняемые параметры (electron-store):**

```typescript
interface AppConfig {
  outputFolder: string;           // последняя выбранная папка
  filenameTemplate: string;       // "timelapse_{date}_{time}"
  fps: number;                    // 0.015625 (1/64) до 1
  ignoreSimilarFrames: boolean;   // включить фильтрацию
  similarityThreshold: number;    // 0-100%, по умолчанию 5%
  codec: 'libx264' | 'libx265';   // H.264 или H.265
  crf: number;                    // 18-28, качество
  captureMode: 'screen' | 'window' | 'area';
  selectedSourceId: string | null;
  hotkeys: {
    startStop: string;            // "F9"
    pause: string;                // "F10"
  };
  minimizeToTray: boolean;
  showNotifications: boolean;
}
```

## 4. Этапы разработки

### Этап 3: Захват экрана
- Интеграция `desktopCapturer`
- Выбор источника, превью
- Сохранение выбора в конфиг

### Этап 4: FFmpeg интеграция
- Загрузка бинарников для всех платформ
- Обертка FFmpegService
- Тестовая запись видео

### Этап 5: Движок записи 
- Таймер с точным FPS
- Буферизация кадров

## 5. Особенности реализации Time Lapse

### Проблема: Низкий FPS в FFmpeg

**Решение:** Использовать `image2pipe` или промежуточные файлы

```typescript
// При FPS < 1 лучше сохранять кадры отдельно
private async captureFrame(): Promise<Buffer> {
  const sources = await desktopCapturer.getSources({
    types: ['screen'],
    thumbnailSize: { width: 1920, height: 1080 }
  });
  return sources[0].thumbnail.toPNG();
}

// Сборка в конце записи
private async finalizeRecording(): Promise<void> {
  await this.ffmpeg.exec([
    '-framerate', String(this.fps),
    '-i', 'frame_%04d.png',
    '-c:v', 'libx264',
    '-pix_fmt', 'yuv420p',
    'output.mp4'
  ]);
}
```

### Проблема: Сравнение кадров в JS медленное

**Решение:** Делегировать FFmpeg

```typescript
// FFmpeg сам фильтрует похожие кадры
const filter = `select=gt(scene\\,${threshold/100})`;
await this.ffmpeg.exec([
  '-i', 'input.mp4',
  '-vf', filter,
  '-vsync', 'vfr',
  'output.mp4'
]);
```

| Платформа | Особенность | Решение |
|-----------|-------------|---------|

## 7. Производительность и оптимизации

| Проблема | Решение |
|----------|---------|
| Утечка памяти при долгой записи | Потоковая запись в FFmpeg, не хранить кадры |
| Блокировка UI при кодировании | FFmpeg в отдельном `worker_threads` |
| Большой размер приложения | Сжатие бинарников FFmpeg, удаление ненужных кодеков |
| Задержка при старте записи | Предварительный "прогрев" FFmpeg |
